# Red Team Command Doctrine

## Purpose
Use this skill when red-team mode is active and the task needs offensive discipline without heavy prompt injection.

## Use pattern
1. Identify the current phase.
2. Select the right method before selecting the technical path.
3. Route into the most specific router and then the most specific leaf skill.
4. Open only the matching references needed for the current path.
5. If the task is large or multi-step, switch to structured orchestration instead of free-form planning.
6. End with one concrete next step.
